# **ポートフォリオ仕様書 (Portfolio Specification)**

本プロジェクトは、採用選考における「紙/PDF資料としての提出」と「Web上での技術アピール」という異なるニーズを満たすため、用途別に最適化された2つのHTMLファイルで構成されています。

## **1\. ファイル構成と役割**

| ファイル名 | 用途 | ターゲット | コンテンツ方針 | レイアウト特性 |
| :---- | :---- | :---- | :---- | :---- |
| **print.html** | PDF出力・提出用 | 採用担当者、エージェント | **実名・具体名** (機密情報含む) Markdownの詳細を完全網羅 | **固定レイアウト** A4横 (297mm×210mm) スライド形式 (全8ページ) |
| **index.html** | Web公開用 | 一般公開、技術確認 | **匿名化** (企業名・固有名詞を伏せる) Web閲覧用に最適化 | **レスポンシブ** 縦スクロール モダンUI (Bento Grid等) |

## **2\. 詳細仕様**

### **2.1. print.html (PDF出力用)**

ブラウザの印刷機能を利用して、PowerPointのような高品質なスライド資料を生成するためのファイルです。

* **基本設定:**  
  * **用紙サイズ:** A4 横 (Landscape)  
  * **基準フォントサイズ:** 11pt (視認性重視)  
  * **配色:** print-color-adjust: exact により、背景色や文字色を画面通りに出力。  
* **レイアウト:**  
  * CSS Gridを使用し、コンテンツ量に応じて 1:1, 1:1.5, 1:2 などの比率で分割。  
  * **強制改ページ (page-break-after: always)** により、セクションごとに1枚のスライドとして出力。  
  * フッター領域を確保し、コンテンツとページ番号（例: 01 / 08）の干渉を防止。  
* **コンテンツ制御:**  
  * **表紙 (P.1):** 氏名、連絡先、Webポートフォリオへの導線（QRコード）、本資料の作成技術スタックを配置。  
  * **日本語処理:** BudouX を導入し、単語途中での不自然な改行を防止。  
  * **チャート:** 印刷時のアスペクト比崩れを防ぐため、maintainAspectRatio: false と固定高さ指定を組み合わせた描画ロジックを採用。

### **2.2. index.html (Web公開用)**

フロントエンドエンジニアとしての技術力を証明するための、一般公開用ポートフォリオサイトです。

* **基本設定:**  
  * **ビューポート:** モバイルファーストのレスポンシブデザイン。  
  * **テーマ:** ダークモード対応（システム設定連動 \+ 手動切り替え）。  
* **UI/UXデザイン:**  
  * **Bento Grid:** Apple等の製品ページに見られる、矩形を組み合わせたグリッドレイアウトを採用し、情報を整理。  
  * **Glassmorphism:** ヘッダー等にすりガラス効果 (backdrop-filter) を適用。  
  * **インタラクション:** スクロール連動のアニメーション (IntersectionObserver)、ホバーエフェクト。  
* **コンテンツ制御:**  
  * **プライバシー保護:** 勤務先（株式会社キッズプロモーション → 総合広告代理店）や取引先名を一般名詞に置換。  
  * **同期:** 職務経歴やスキルセットの内容は print.html と整合性を持たせつつ、Webで読みやすい形式（リストや短い段落）に整形。

## **3\. 技術スタック**

両ファイル共通で使用している技術要素です。

* **フレームワーク:** Tailwind CSS (v3.4) \- CDN版  
* **チャートライブラリ:** Chart.js (v4.4)  
* **テキスト整形:** BudouX (日本語分かち書き)  
* **フォント:**  
  * 日本語: Noto Sans JP  
  * 英語見出し/数値: Outfit  
  * コード/補足: JetBrains Mono  
* **アイコン:** Font Awesome (Free CDN)
